■ オーナー明細メール下書きツール（Google Apps Script）

1. オーナー一覧のスプレッドシートを開く
   https://docs.google.com/spreadsheets/d/1j1To2j0eu-7UvmionfabvW4UYHobItiT2Qe84dHT5qg/

2. メニュー「拡張機能」→「Apps Script」でエディタを開き、このフォルダ内の Code.gs の
   内容をすべて貼り付けて保存する。

3. スプレッドシートの1行目に、次の列を用意する（D列が重要）。
   A: 宛名（例: 藤原様）
   B: メール
   C: 施設名
   D: 施設コード（PDFファイル名の英数字部分と完全一致。例: ファイルが 3B202602.pdf なら 3B）

4. エディタで「保存」後、初回は「権限を確認」から Google アカウントで許可する
   （Drive・Gmail・スプレッドシートへのアクセス）。

5. スプレッドシートに戻り、数秒待つとメニューに「明細メール」が出る。
   「下書きを一括作成（ドライブのPDF）」を実行する。

6. 明細 PDF は次のフォルダに置く（デフォルトIDはスクリプト内に記載済み）。
   https://drive.google.com/drive/folders/1qZd-WvHbKHjBVouunSqsWKIiXpsgfx0D
   別フォルダにしたい場合はメニュー「設定: 監視フォルダID」から変更可能。

■ PDF のファイル名（運用で統一）

   推奨: 施設名 + YYYYMM + .pdf（例: THE HILLTOP GARDEN VILLA Can202602.pdf）
   または: 施設コードのみ + YYYYMM（例: 3B202602.pdf）… 台帳「施設コード」列と一致
   旧形式「◯月.pdf」も読み取り可能です。同フォルダの .xlsx は無視。

■ 動作

   - Gmail の「下書き」に、宛先・件名・本文・PDF添付まで作成します。
   - 毎回 CC に k-aoki@llc-blueocean.jp が入ります（変更は Code.gs の DRAFT_CC）。
   - 「送信」は行いません。担当者が Gmail で確認してから送信してください。

■ clasp を使う場合（任意）

   このリポジトリの gas フォルダをプロジェクトとして push しても構いません。

■ Gmail → kintone 連携（コード.js）

   API トークンは Git に書かず、Apps Script の「スクリプトプロパティ」だけに保存する。

   プロジェクト設定 → スクリプトプロパティ に次を登録:

     KINTONE_SUBDOMAIN   … 例: diyso2o309q7.cybozu.com
     KINTONE_APP_ID      … 例: 12
     KINTONE_API_TOKEN   … kintone の API トークン（コード・チャット・メールに長期保存しない）
     （省略可）GMAIL_LABEL_NAME … 既定は Kintone転送待ち
     （省略可）GMAIL_SUBJECT_CONTAINS … 既定は「新規予約が入りました」（「！！！」の有無は問わず部分一致）

   任意: MAX_THREAD_AGE_HOURS に数値（例: 24）で、古いスレを処理しない。

   任意: BODY_USE_LAST_MESSAGE に 1 … 本文だけスレッド「最新」の 1 通から取る。
         省略時は「先頭」＝予約通知の本文（返信で金額がずれるのを防ぐ）

   任意: kintone の「フィールドコード」と既定が違うときだけ設定
     FIELD_LODGING（既定 auto_room） FIELD_PLATFORM（既定 auto_platform）
     FIELD_CHECKIN（既定 checkin） FIELD_CHECKOUT（既定 checkout）
     ほか guest_name / commission 等も同様に FIELD_* で上書き可（コード.js 内 getFieldCodes_ 参照）

   宿泊施設が「ドロップダウン」のとき（API には一覧の選択肢と完全一致する文字列が必要）:
     LODGING_APPEND_ROOM に 0 … 「施設名 / 部屋」の連結をやめる（施設だけを FIELD_LODGING に送る）
     FIELD_ROOM_NAME … 部屋タイプを入れる別フィールドのフィールドコード（フォームにあれば）
     FACILITY_OPTION_MAP_JSON … 例:
       {"Hilltop Garden Villa CAN":"プルダウンに表示されている施設名と同じ文字列"}
       メールの表記とプルダウンの表記が違うときに必須

   予約サイトが「ドロップダウン」のときも同様に、一覧と完全一致する文字列が必要:
     PLATFORM_OPTION_MAP_JSON … 例:
       {"Airbnb":"Airbnb"}
       メールに「Airbnb」、一覧が別表記のときにキー→値で対応付ける

   任意: DEBUG_LOG_RECORD に 1 … 宿泊施設・予約サイトのフィールドコードと値を別行で出力し、続けて record 全体も出力（確認後は削除推奨）

   初回は gmailToKintone を手動実行して権限を許可後、
   installHourlyTrigger または installTriggerEveryMinutes(15) で時間トリガーを作成。

■ うまく入らないとき（確認順）

   1. DEBUG_LOG_RECORD=1 で実行し、ログの DEBUG lodging / DEBUG platform の value を見る
      → 空ならメールから未取得かフィールドコード不一致。値があるのに画面が空ならプルダウン完全一致か。

   2. kintone のフォームで「宿泊施設」「予約サイト」のフィールドコードが
      FIELD_LODGING / FIELD_PLATFORM（既定 auto_room / auto_platform）と一致するか

   3. プルダウンのとき … メールの表記と一覧の表記が一字一句同じか。
      違う場合は FACILITY_OPTION_MAP_JSON / PLATFORM_OPTION_MAP_JSON

   4. LODGING_APPEND_ROOM=0 … 施設名だけを送る（「施設 / 部屋」はドロップダウンと相性が悪い）

   5. 金額がずれるとき … BODY_USE_LAST_MESSAGE は通常オフ（先頭メール＝予約本文）。
      返信後の最新だけ読みたいときだけ 1
