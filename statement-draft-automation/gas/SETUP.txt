■ オーナー明細メール下書きツール（Google Apps Script）

1. オーナー一覧のスプレッドシートを開く
   https://docs.google.com/spreadsheets/d/1j1To2j0eu-7UvmionfabvW4UYHobItiT2Qe84dHT5qg/

2. メニュー「拡張機能」→「Apps Script」でエディタを開き、このフォルダ内の Code.gs の
   内容をすべて貼り付けて保存する。

3. スプレッドシートの1行目に、次の列を用意する（D列が重要）。
   A: 宛名（例: 藤原様）
   B: メール
   C: 施設名
   D: 施設コード（PDFファイル名の英数字部分と完全一致。例: ファイルが 3B202602.pdf なら 3B）

4. エディタで「保存」後、初回は「権限を確認」から Google アカウントで許可する
   （Drive・Gmail・スプレッドシートへのアクセス）。

5. スプレッドシートに戻り、数秒待つとメニューに「明細メール」が出る。
   「下書きを一括作成（ドライブのPDF）」を実行する。

6. 明細 PDF は次のフォルダに置く（デフォルトIDはスクリプト内に記載済み）。
   https://drive.google.com/drive/folders/1qZd-WvHbKHjBVouunSqsWKIiXpsgfx0D
   別フォルダにしたい場合はメニュー「設定: 監視フォルダID」から変更可能。

■ PDF のファイル名（運用で統一）

   推奨: 施設名 + YYYYMM + .pdf（例: THE HILLTOP GARDEN VILLA Can202602.pdf）
   または: 施設コードのみ + YYYYMM（例: 3B202602.pdf）… 台帳「施設コード」列と一致
   旧形式「◯月.pdf」も読み取り可能です。同フォルダの .xlsx は無視。

■ 動作

   - Gmail の「下書き」に、宛先・件名・本文・PDF添付まで作成します。
   - 毎回 CC に k-aoki@llc-blueocean.jp が入ります（変更は Code.gs の DRAFT_CC）。
   - 「送信」は行いません。担当者が Gmail で確認してから送信してください。

■ clasp を使う場合（任意）

   このリポジトリの gas フォルダをプロジェクトとして push しても構いません。

■ Gmail → kintone 連携（コード.js）

   API トークンは Git に書かず、Apps Script の「スクリプトプロパティ」だけに保存する。

   プロジェクト設定 → スクリプトプロパティ に次を登録:

     KINTONE_SUBDOMAIN   … 例: diyso2o309q7.cybozu.com
     KINTONE_APP_ID      … 例: 12
     KINTONE_API_TOKEN   … kintone の API トークン（コード・チャット・メールに長期保存しない）
     （省略可）GMAIL_LABEL_NAME … 既定は Kintone転送待ち
     （省略可）GMAIL_SUBJECT_CONTAINS … 既定は「新規予約が入りました」（「！！！」の有無は問わず部分一致）

   任意: MAX_THREAD_AGE_HOURS … 先頭メールの受信から指定時間を超えたスレはスキップ。
         深夜の予約が入らないときは、このプロパティを削除するか値を大きくする（誤設定で弾かれていることがある）

   任意: BODY_USE_LAST_MESSAGE に 1 … 本文だけスレッド「最新」の 1 通から取る。
         省略時は「先頭」＝予約通知の本文（返信で金額がずれるのを防ぐ）

   任意: kintone の「フィールドコード」と既定が違うときだけ設定
     FIELD_LODGING（既定 lodging2 …【必須】宿泊施設2）
     FIELD_PLATFORM（既定 platform2 …【必須】予約サイト2）
     FIELD_ROOM_NAME（既定 room_name2 …【必須】部屋名2）
     FIELD_CHECKIN（既定 checkin） FIELD_CHECKOUT（既定 checkout）
     ほか guest_name / commission 等も同様に FIELD_* で上書き可（コード.js 内 getFieldCodes_ 参照）

   メールからは「宿泊施設」「予約サイト」「部屋タイプ」をそれぞれ別の文字列フィールドへ送る（連結しない）。

   任意: FACILITY_OPTION_MAP_JSON / PLATFORM_OPTION_MAP_JSON … メール表記を別の文字列に変換したいとき

   任意: DEBUG_LOG_RECORD に 1 … 宿泊施設・予約サイト・部屋のフィールドコードと値を別行で出力し、続けて record 全体も出力（確認後は削除推奨）

   初回は gmailToKintone を手動実行して権限を許可後、
   installHourlyTrigger（15分間隔）または installTriggerEveryMinutes(15) で時間トリガーを作成。

■ Gmail フィルタ（件名に「新規予約が入りました」→ ラベル「Kintone転送待ち」を自動付与）

   ブラウザで Gmail を開き、次のとおり設定する（GAS からはフィルタを作れないため手動作業）。

   1. 左サイドバーで「ラベルを表示」→ 下の「作成」を押し、ラベル名を
      「Kintone転送待ち」で作成（既にあれば省略）。

   2. Gmail 右上の歯車 →「すべての設定を表示」→ 上部タブ「フィルタとブロック中のアドレス」
      →「新しいフィルタを作成」。

   3. 「件名」欄に次を入力（部分一致でよいので「！！！」は含めなくてよい）:
        新規予約が入りました
      「検索オプション」で件名のみにしたい場合は、検索窓で
        subject:新規予約が入りました
      と入力してから「フィルタを作成」でも可。

   4. 「フィルタを作成」をクリックし、次の画面で
      「ラベルを付ける」にチェック →「Kintone転送待ち」を選択。
      （必要なら「受信トレイをスキップ」は付けない／運用に合わせる）

   5. 「フィルタを作成」で保存。

   これで該当件名のメールが届くたびにラベルが付き、時間トリガーで GAS が kintone 登録する。

■ うまく入らないとき（確認順）

   1. DEBUG_LOG_RECORD=1 で実行し、ログの DEBUG lodging / DEBUG platform / DEBUG room の value を見る

   2. kintone のフォームで「宿泊施設2」「予約サイト2」「部屋名2」のフィールドコードが
      lodging2 / platform2 / room_name2 と一致するか（違う場合は FIELD_* で上書き）

   3. 値がログにあるのに画面が空 … 別のフィールドを見ていないか、フィールドコードの打ち間違いを確認

   4. 深夜だけ入らない … スクリプトプロパティ MAX_THREAD_AGE_HOURS を削除または十分大きくする。
      実行ログに「スキップ（経過時間」が出ていないか確認。

   5. ログに「Kintone転送待ちスレッド数: 0」なのに Gmail にラベル付きがある …
      フィルタの付与先アカウントと GAS が動く Google アカウントが同一か確認。
      ラベル付きが500件超で取りこぼしていた場合は、コードがページング取得するよう修正済み（clasp push 要）。

   6. 金額がずれるとき … BODY_USE_LAST_MESSAGE は通常オフ（先頭メール＝予約本文）。
      返信後の最新だけ読みたいときだけ 1
